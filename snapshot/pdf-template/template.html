Section 2

Security Headers Audit

HTTP security headers are the first line of defense for modern websites. They tell visitor browsers how to behave when loading and interacting with your pages. A missing header is not always an immediate vulnerability, but stacked missing headers create attack surface that adversaries exploit routinely. We check for the Core 6 headers plus 4 optional ones.

Header Status Value / Impact
Strict-Transport-Security Pass max-age=63072000 (2 years). Prevents SSL downgrade attacks on returning visitors.
Content-Security-Policy Pass default-src 'self' with explicit allowlist for Google Fonts. Blocks inline script injection and third-party resource loading.
X-Frame-Options Pass DENY. Prevents clickjacking attacks via iframe embedding.
X-Content-Type-Options Pass nosniff. Prevents MIME-type confusion attacks.
Referrer-Policy Pass strict-origin-when-cross-origin. Limits information leakage in outbound referrer headers.
Permissions-Policy Pass camera, microphone, geolocation, payment all disabled. Browser API gate for privacy-sensitive features.

All 6 core headers pass.

babakizo.com has the strongest security header configuration we have observed on a personal Nigerian site in 2026. This is a reference configuration worth copying. The CSP in particular is specific rather than permissive, which is rare.

Optional headers (not checked in the core scoring)

Cross-Origin-Opener-Policy, Cross-Origin-Embedder-Policy, Cross-Origin-Resource-Policy, and Expect-CT are not part of the Securva core 6 because they are either deprecated (Expect-CT) or situationally-applied (COOP/COEP/CORP need specific use cases). babakizo.com does not set these and does not need to at its current scope.

Nigeria Data Protection Act (NDPA) Compliance

Under the Nigeria Data Protection Act 2023, any website that handles personal data of Nigerian residents is a data controller and must meet specific technical and organizational standards. The Nigeria Data Protection Commission (NDPC) has been actively enforcing NDPA in 2025 and 2026, with fines of up to NGN 10 million or 2% of gross annual revenue for violations.

Section 8

Remediation Roadmap

This section normally lists prioritized fixes based on findings. For babakizo.com, the good news is there are no critical or high-severity findings to remediate. Everything below is aspirational or preparatory for future changes.

Priority: Low (no urgency)

  • Add a CAA DNS record restricting certificate issuance to Let's Encrypt only. Prevents a rogue CA from issuing a fraudulent certificate for your domain. Takes 30 seconds in Cloudflare DNS.
  • Add a SPF + DMARC record even though you do not send email. This prevents spammers from spoofing your domain in email. Set SPF to v=spf1 -all and DMARC to v=DMARC1; p=reject. Protects your brand reputation.
  • Subscribe to Securva Continuous Monitoring for automatic alerts on certificate rotation, security grade regression, and new CVE matches against your tech stack. NGN 15,000/month.

Priority: If babakizo.com transitions to commercial use

  • Document a Data Protection Officer contact on the privacy policy. Required under NDPA for commercial data processing.
  • Add formal data subject rights request process (access, rectification, erasure, portability). Required under NDPA.
  • Publish cross-border transfer disclosure in the privacy policy. Required because Cloudflare is a US-based CDN.
